# 远程程控精选 40 例数据集盘点与系统迁移评估

更新时间：2026-07-14

## 1. 记录目的与边界

本记录用于后续对话快速恢复数据集知识，并评估这批真实数据能否接入项目现有的
“音频 → ASR noisy transcript + 置信度 → 医生交互确认 → confirmed transcript →
病例信息整理与鲁棒性评估”流程。

数据源已于 2026-07-14 迁入项目内受 Git 忽略规则保护的本地 raw 目录：

```text
data/raw/remote_programming_40/远程程控人工复核资料_精选40例_无病历版_20260713/
```

本文件只记录目录结构、聚合统计和迁移问题，不记录病例原文、患者姓名、音频内容、
逐例样本映射或其他可识别信息。原始音频和逐例转录可以保存在本地工作区的 ignored data
目录中，但不得加入 Git、公共仓库、论文评审包或未经批准的存储位置。

授权状态补充：2026-07-14 用户明确确认本批数据的外部 LLM、外部 ASR 和原音频处理范围
均已经获审。因此在线模型与云端转写不是当前阻塞，授权范围不再作为迁移问题。
实验仍应按常规记录服务商、模型、输入字段和运行配置，并遵守项目规则，不把患者原文、
音频或未脱敏派生数据提交到 Git。

## 2. 数据集定位

这批资料应定位为：

> 真实远程程控音频及多路自动转录的内部人工复核包。

它不是已经完成标注的 clean/reference 数据集，也不是已经由医生确认的
confirmed transcript 数据集。“无病历版”只表示包内没有整理病历，不表示音频和转录
已经彻底去标识化，也不表示自动转录可作为真值。

根级说明同时提醒：所有转录都可能有错字、漏字、幻觉和说话人/来源误判，必须结合录音
核对；商业转录供应商尚未核实，不能直接标记为讯飞。根级通用说明原本写有不得上传外部
平台，本项目以用户 2026-07-14 对本批数据全流程已经获审的明确确认为准。

用户转述的生成链为：

> 带医学先验的 Qwen ASR 转录 → 本地 Qwen 27B → 自动病历 Markdown。

该包共 40 份 `病例_XXXX_Qwen病历_v1.7.5.md`，匿名病例编号与 20260713 无病历版
40/40 完全一致；另有 1 个 `.DS_Store`。`v1.7.5` 目前只按文件中的产物版本标签记录，
不能在没有上游配置的情况下解释为具体模型或 prompt 版本。

40 份 Markdown 合计约 17.6 KiB，正文平均约 166 字符，最短 11、最长 341 字符。栏目不是
每例固定齐全：处置 38 例、现病史 29 例、随访计划 20 例、查体 8 例、主诉 6 例、
既往史/过敏史 4 例；20/40 文件含“需核实”，4 份只有一个栏目。它更像面向远程程控的
简短自动病例整理，而不是完整、人工定稿的电子病历。

系统定位：

- 它是现成的 `noisy_asr → case_summary` 下游自动输出 baseline；
- 它不是新的 ASR transcript，不参与 ASR 置信度、时间戳或候选构建；
- 它不是人工 clean/reference、真实 confirmed transcript 或下游 gold；
- 它会同时继承带医学先验 Qwen ASR 的识别误差，以及 Qwen 27B 的遗漏、归纳偏差或生成
  错误，不能倒灌到上游当作转录真值；
- P0 可将其用作病例整理格式参考和既有输出对照。若以后做质量比较，应固定同一个
  Qwen 27B、prompt/schema 和解码配置，分别输入原始 Qwen ASR 与 confirmed transcript；
  当前还缺精确 Qwen ASR/27B 模型标识、prompt、解码参数和运行配置，不能直接复现。

## 4. 已发现的数据质量信号

### 4.1 说话人与信息来源

根级清单推荐的说话人数 `K` 为 2–6：

| K | 病例数 |
|---:|---:|
| 2 | 13 |
| 3 | 9 |
| 4 | 7 |
| 5 | 6 |
| 6 | 5 |

推荐转录共有 5,942 个自动时间片段。自动角色分布为：医生 3,379、患者 1,325、
未知 993、家属/照护者 224、工作人员 19、无关内容 2。约 16.7% 片段角色仍为未知；
且这些角色来自自动流程，不可当作 gold speaker role。

这与 PriMock57 固定 doctor/patient 双通道结构差异很大。后续至少要支持医生、患者、
家属/照护者、工作人员、未知和无关内容，并把“声学聚类说话人”和“语义角色”分开记录。

### 4.2 时间片段

- 5,942 个自动片段；单例最少 12、中位 108、最多 540；
- 片段时长中位约 2.16 秒；
- 有 187 个零时长片段，需要回听并修正或标记无效；
- 所有病例的相邻语音片段之间都有间隔，这主要可能是静音，不应直接视为漏转；
- 5 例推荐转录的最后一个片段距音频结尾超过 10 秒，最大约 41.36 秒，需要确认是尾部静音还是缺失；
- 商业转录只提供片段起点，不能直接构造可靠的词级回听窗口。

### 4.3 “高置信/待核”统计的可追溯性

根级 CSV 给出以下聚合字段：

- 证据文本字符数代理值合计 89,128；
- 高置信文本字符数合计 22,207；
- 高置信片段数合计 889；
- 待核医疗片段数合计 1,629。

但是逐例推荐转录中没有导出相应的置信度值、颜色级别、阈值、触发原因或待核片段 ID，
数据包内也没有运行配置、prompt、模型权重版本、解码参数或校准记录。因此这些字段目前
只能当作上游流程的代理统计，不能直接接到项目的 `ASRConfidenceRecord`，也不能替代
ASR 原生 token/word posterior 或经过 reference 校准的风险概率。

### 4.4 多模型输出一致性与疑似失败

- SenseVoice 的“有医疗先验/无医疗先验”在 40/40 例中完全相同，说明当前先验设置
  对该路输出没有产生实际变化，不能把两份文件当作独立证据；
- Paraformer 两版长度几乎相同，但 40 例均非逐字完全一致，需要进一步判断差异是标点、
  术语替换还是后处理；
- Qwen3-ASR 两版的长度比范围约 0.456–27.425，Whisper-large-v3 约 0.049–1.800，
  存在明显截断、空转或提示导致输出膨胀的风险；
- 80 份 Qwen3-ASR 当前输出中有 6 份少于 500 字符；80 份 Whisper-large-v3 中有
  10 份少于 500 字符、2 份少于 100 字符；另有 1 份旧 Whisper-large 少于 100 字符。

在把多模型文本用作候选或一致性信号前，需要做长度异常、重复、语言、幻觉、覆盖率和
时间对齐 QC，不能假设每一路都是完整转录。

## 6. 工程跑通与后续评估的分层

### P0-1 外部模型不是阻塞

用户已确认外部 LLM、外部 ASR 和原音频处理均已获审。本项目不再把外部调用授权或隐私
筛查列为迁移问题，只按实验复现要求记录服务商、endpoint、模型版本、输入字段和运行配置。

### P2-1 人工 clean/reference（后续评估，不阻塞 P0）

包内所有转录都是模型结果或模型后处理结果，不能把商业转录或“推荐转录”当作
clean/reference。至少需要对一个代表性 pilot 子集由人工结合原音频制作：

- 带时间戳的逐字/逐句 reference；
- 说话人聚类到医生、患者、家属等角色的人工映射；
- 药名、剂量、DBS 参数、数字单位、侧别、否定词和时间信息的重点复核；
- 不确定/听不清标记和双人仲裁规则。

没有 reference 时可以跑通并演示完整工作流，但不能校准置信度、计算 WER/CER、评估
top-k 覆盖率，也不能可靠比较 noisy 与 confirmed transcript。用户当前明确要求先跑通，
所以本项推迟到质量评估阶段。

### P0-2 缺少中文 ASR 时间戳、风险信号和候选接口

现有多路纯文本不能支持词级绿/黄/红、逐词局部回听和同解码器 top-k。P0 需要从
Parakeet CTC 直接导出 token/span/segment 时间戳、帧分布派生风险信号、解码配置和
sequence n-best；若回退云端，则对应字段允许使用可空 provider 风险分数和 alternatives。
没有 reference 时先用明确标为 `calibrated=false` 的分位数工程阈值；以后评估时再重新校准。
PriMock57 + 英文 NeMo 的阈值和错误率不能跨模型、跨语言、跨场景迁移。

### P0-3 中文处理链路不兼容

当前 `asr_transcript.split()`、英文医学关键词、英文边界词和按空格回放的实现都会在中文
上失效。若不先修复，界面可能只显示一个超长 token，候选无法绑定到局部片段，confirmed
transcript 还会被错误加空格。

### P2-2 下游 gold 层（后续评估，不阻塞 P0）

“无病历版”没有可作为摘要真值的整理病历。要评估症状抽取、病例信息整理和程控计划，
若要评估，需要由独立标注者从人工 reference 构建结构化 gold facts；不能让生成摘要的同一模型自动
生成 gold。远程程控任务还应新增 DBS 专用事实类型，例如设备/侧别、触点、幅度或电压/
电流、脉宽、频率、开关状态、参数调整前后、症状变化、不良反应和随访计划，并严格区分
患者自述、医生提问、医生判断和医生建议。

## 7. 建议迁移顺序

### 阶段 A：数据快照与实验配置

1. 冻结 `20260713` 快照并记录文件数量、大小和校验摘要，避免实验中静默变化；
2. 在项目内建立匿名 manifest 和路径指针，不把原音频或逐例转录提交到 Git；
3. 在运行配置中记录 ASR/LLM 服务商、endpoint、模型版本、输入字段和解码参数。

### 阶段 B：1 例中文工程 smoke test

本地现有 artifact：
`data/external/asr_models/nemo/Parakeet-Hybrid-XL-unified-0.6b_spe7k_zh-en-CN_3.0.nemo`。
先记录来源、许可、digest，并在 WSL `clinical-asr` / NeMo 3.1.0 做 restore-only，确认其
真实模型类和解码配置；再将 1 个 MP3 确定性解码为 16 kHz mono
PCM/WAV，以 15–30 秒片段、`batch_size=1`、FP16/BF16 验证中文/英文/中英混说文本、
CTC token 时间、帧 entropy/margin、稀疏 top-k 和 sequence n-best。当前 RTX 4060 只有
8 GB，不能沿用 120 秒窗口假设，也不在第一步部署完整 Riva/NIM。直接 NeMo 不可行时再
回退 Google Chirp 2。详细调研见 `docs/t050_chinese_asr_flow_research.md`。

### 阶段 C：中文 ASR 置信度最小闭环

1. 保留原 MP3，另生成可追溯的 PCM/WAV ASR 输入，并保留 VAD/窗口到原音频的绝对偏移；
2. 导出 CTC/SentencePiece 单元、时间戳、entropy/margin 和 beam/4-gram LM sequence n-best；
3. 将 schema、页面拼接和回放逻辑改为 CTC/subword unit + 字符偏移，不依赖空格；
4. 用 `demo_quantile_v0` 工程阈值产生绿/黄/红，明确 `calibrated=false`；
5. 分开保存 `am_raw`、`decoded_with_lm`、`display_itn`，记录文本变换对齐；
6. 后接 Silero VAD/Sortformer，并按时间重叠复用现有自动角色；扩展中文/DBS 实体 gating，
   重建审阅页并人工点击验收；
7. 按 1 → 5 → 40 例扩量，导出 feedback、confirmed transcript 和病例信息整理结果。

### 阶段 D：confirmed transcript 与下游工程贯通

1. 审阅风险片段，形成 feedback log 和 confirmed transcript；
2. 将 confirmed transcript 送入中文病例信息整理链路；
3. 检查反馈可重复回放、中文不被错误插空格、局部回听可 seek、结构化输出可生成；
4. 本阶段不计算正确率或质量提升。

### 阶段 E：质量评估（当前推迟）

需要证明质量提升时，再选代表性子集制作人工 clean/reference、人工角色和下游 gold，计算
CER、医学实体/数字错误、置信度校准、top-k 覆盖率、审阅成本及下游差异。

## 8. 当前结论

这批 40 例非常适合作为项目从公开英文模拟数据迁移到真实中文远程程控场景的目标数据，
尤其适合研究多参与者、医学术语、DBS 参数、数字单位、说话人/信息来源和人工回听成本。
外部 LLM、外部 ASR 和原音频处理均已获审，在线模型不是阻塞。对当前“先跑通”目标，
真正的 P0 阻塞是中文 ASR 接口、中文字符级回放、可空风险分数/候选和多角色 schema；
人工 reference 与独立下游 gold 只是以后证明质量提升时才需要的评估条件。

在补齐这些条件前，这批资料应按受保护的 raw audio + noisy/auxiliary transcript 集合使用，
不能宣称已经完成端到端质量提升评估。
